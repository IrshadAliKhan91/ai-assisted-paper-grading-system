# Contributing to FairMark

Thank you for your interest in contributing to FairMark - the AI-Powered Paper Checking System! Below are guidelines to help you get started.

## How to Contribute

1. **Fork the Repository**: Create a personal copy of the repository on GitHub.
2. **Clone the Fork**: Clone your fork to your local machine.
3. **Create a Branch**: Create a branch for your changes (e.g., `feature/awesome-new-feature` or `bugfix/issue-description`).
4. **Make Changes**: Commit your changes with clear, descriptive commit messages.
5. **Push & Pull Request**: Push your branch to GitHub and open a Pull Request (PR) to the `main` branch.

## Development Setup

To set up the development environment, configure the database, and run the backend/frontend services locally, please follow the step-by-step instructions in [SETUP.md](file:///SETUP.md).

## Code Style Guidelines

- **Python (Backend)**: Follow PEP 8 style standards. Write clean, modular, and self-documenting code.
- **JavaScript (React Frontend)**: Keep components reusable and styles organized. Follow standard ES6+ syntax.
- **Documentation**: Keep comments and markdown documents updated with any code changes.

## Pull Request Process

- Ensure all existing unit tests in the `backend` pass before submitting.
- Avoid introducing unnecessary or temporary files to the repository (add them to your local git ignore if needed).
- Write a clear description in your Pull Request explaining what changes were made and why.
